# Code of Conduct

## Our commitment

We want Verdkomunumo Flutter to be a respectful, technically rigorous, and welcoming open source project.

Participants are expected to:

- be respectful and professional
- assume good intent, but discuss technical disagreements directly
- focus criticism on code, architecture, and behavior rather than people
- avoid harassment, discrimination, or hostile conduct

## Expected behavior

- Use clear and constructive language.
- Share context when raising bugs, regressions, or concerns.
- Respect maintainer decisions on architecture, scope, and quality gates.
- Keep discussions aligned with the repository documentation in `docs/`.

## Unacceptable behavior

- Personal attacks or insulting language
- Harassment or discrimination
- Repeated off-topic disruption
- Publishing private information without permission
- Deliberately bypassing project standards after review feedback

## Enforcement

Maintainers may edit, remove, or reject comments, issues, PRs, or contributions that violate this code of conduct.

Repeated or severe violations may result in temporary or permanent exclusion from the project.

## Reporting

If you need to report unacceptable behavior, open a private maintainer contact through the repository security or maintainer channels rather than escalating publicly.

## Scope

This code of conduct applies to repository discussions, issues, pull requests, review comments, and other project-managed spaces.
