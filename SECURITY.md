# Security Policy

## Supported versions

Security fixes are only guaranteed for the latest state of the default branch until an explicit release support matrix is introduced.

## Reporting a vulnerability

Do not open public GitHub issues for suspected security vulnerabilities.

Instead:

1. Prepare a concise report with impact, affected area, reproduction details, and any suggested mitigation.
2. Send it privately to the project maintainers through the repository security contact or maintainer channel.
3. Allow reasonable time for triage and remediation before public disclosure.

## Response expectations

Maintainers will aim to:

- acknowledge receipt
- assess severity and scope
- prepare a fix or mitigation
- communicate disclosure timing if a coordinated release is required

No SLA is guaranteed yet, but reports that are clear and reproducible will be prioritized.
